Gestion des Utilisateurs & Groupes

New-LocalUser Créer un compte utilisateur local sécurisé.
Note : Pour manipuler les mots de passe, PowerShell nécessite l'utilisation d'une chaîne sécurisée (SecureString).
# Méthode interactive
New-LocalUser -Name "Admin_Temp" -Description "Compte temporaire"-NoPassword

# Avec mot de passe prédéfini
$Password = Read-Host -AsSecureString
: **********
New-LocalUser -Name "Alice" -Password $Password -FullName "Alice Smith"
Add-LocalGroupMember Gérer les privilèges (ex: ajouter aux Administrateurs). 
# Ajouter un utilisateur au groupe local des Administrateurs
Add-LocalGroupMember -Group "Administrateurs" -Member "Alice"

# Vérifier les membres d'un groupe
Get-LocalGroupMember -Group "Utilisateurs du Bureau à distance"

Réseau & Connectivité

Get-NetIPConfiguration L'alternative moderne à ipconfig. 
# Voir la configuration IP détaillée de toutes les interfaces
Get-NetIPConfiguration

# Filtrer pour n'avoir que l'adresse IPv4 d'une interface spécifique
Get-NetIPAddress -InterfaceAlias "Ethernet0" -AddressFamily IPv4
Test-NetConnection Outil de diagnostic réseau ultime (Ping + Port scan).
Pour tester l'ouverture des ports.
# Test de connectivité simple (Ping)
Test-NetConnection -ComputerName "google.com"

# Vérifier si le port RDP (3389) est ouvert
Test-NetConnection -ComputerName "SRV-CORE-01" -Port 3389
New-NetIPAddress Configurer une adresse IP statique.
Explication : Cette commande remplace la configuration DHCP par une configuration statique. Elle nécessite l'Index de l'interface (InterfaceIndex).
# 1. Identifier l'Index de la carte
Get-NetAdapter

# 2. Assigner l'IP et la Passerelle
New-NetIPAddress -InterfaceIndex 12 -IPAddress "192.168.1.50" -PrefixLength 24 -DefaultGateway "192.168.1.1"

# Variante avec Alias
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress "10.0.0.10" -PrefixLength 8
Set-DnsClientServerAddress Définir les serveurs DNS.
Note : Indispensable pour joindre un domaine Active Directory. Vous pouvez passer plusieurs IPs dans une liste séparée par des virgules.
# Configurer un DNS primaire et secondaire
Set-DnsClientServerAddress -InterfaceIndex 12 -ServerAddresses ("1.1.1.1","8.8.8.8")

# Réinitialiser pour utiliser le DHCP
Set-DnsClientServerAddress -InterfaceIndex 12 -ResetServerAddresses
netstat -an Montre les ports ouverts/actifs

# récupérer toutes les connexions IPv4 actives :
netstat -p IP 


# Listes les ports ouverts / actifs
netstat-an

New-NetRoute / Get-NetRoute Gérer la table de routage statique.
Explication : Utilisé pour diriger le trafic vers des réseaux spécifiques via des passerelles précises.
# Ajouter une route vers un réseau distant
New-NetRoute -DestinationPrefix "172.16.0.0/16" -InterfaceIndex 12 -NextHop "192.168.1.254"

# Voir toutes les routes actives (équivalent de route print)
Get-NetRoute -AddressFamily IPv4
Set-NetAdapter / Rename-NetAdapter Administrer les cartes physiques. 
# Renommer une carte pour plus de clarté
Rename-NetAdapter -Name "Ethernet 1" 
-NewName "LAN_Prod"

# Désactiver/Activer une carte
Disable-NetAdapter -Name "LAN_Prod" 
-Confirm:$false
Enable-NetAdapter -Name "LAN_Prod"
Remove-NetIPAddress Supprimer proprement une configuration. 
# Supprimer une adresse IP spécifique
Remove-NetIPAddress -IPAddress "192.168.1.50" -Confirm:$false

# Supprimer toutes les IPs d'une interface (Nettoyage avant reconfiguration)
Get-NetIPAddress -InterfaceIndex 12 | Remove-NetIPAddress -Confirm:$false
Tester la connectivité pas à pas en suivant le modèle OSI du bas vers le haut. 
# Test 1 — Boucle locale (couche 3) : vérifiez que la pile TCP/IP fonctionne sur la VM.
ping 127.0.0.1
# Test 2 — Carte réseau locale : pingez votre propre adresse IP.
ping 192.168.99.50
# Test 3 — Passerelle : vérifiez l'accès au routeur.
ping 192.168.99.2
# Test 4 — Réseau externe : testez l'accès à Internet.
ping 8.8.8.8
# Test 5 — Résolution DNS : vérifiez que la résolution de noms fonctionne.
Resolve-DnsName -Name www.microsoft.com
# Test avancé — Test-NetConnection :
Test-NetConnection -ComputerName 8.8.8.8 -Port 53
Rappel Sécurité : Lorsque vous configurez un serveur à distance via SSH ou WinRM, soyez prudent avec Remove-NetIPAddress ou Disable-NetAdapter, car vous risquez de couper votre propre session de gestion.

Services & Rôles Serveur

Get-Service / Start-Service Contrôler les démons Windows. 
# Lister les services dont le nom contient "sql"
Get-Service -Name "*sql*"

# Redémarrer le service Spouleur d'impression
Restart-Service -Name "Spooler" -Force
Install-WindowsFeature Installer des rôles (IIS, DNS, ADDS).
Uniquement disponible sur les éditions Windows Server.
# Installer le serveur Web IIS avec les outils de gestion
Install-WindowsFeature -Name Web-Server -IncludeManagementTools

# Lister tous les rôles déjà installés sur le serveur
Get-WindowsFeature | Where-Object { $_.Installed -eq $True }

Maintenance Système

Get-EventLog / Get-WinEvent Analyser l'Observateur d'événements. 
# Afficher les 20 dernières erreurs du journal Système
Get-EventLog -LogName System -EntryType Error -Newest 20

# Chercher des événements liés aux échecs de connexion (Security)
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625}
Conseil de Pro : Utilisez toujours l'auto-complétion avec la touche Tab et n'hésitez pas à abuser de Get-Help Nom-De-La-Commande -Examples pour voir des cas d'utilisation concrets.
