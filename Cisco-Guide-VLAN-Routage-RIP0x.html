Partie 1 — Réseau à 3 VLANs sur 2 routeurs

Étape 1 : Créer les VLANs sur les switchs
Configurer plusieurs VLANs, à copier les switchs concernés.





Étape 2 : Affecter les ports des PC aux VLANs

Associer chaque port utilisateur au VLAN correspondant.
configure terminal
interface fa0/1
 switchport mode access
 switchport access vlan 10
interface fa0/2
 switchport mode access
 switchport access vlan 20
interface fa0/3
 switchport mode access
 switchport access vlan 30
end
write memory


Étape 3 : Configurer le port trunk vers le routeur
Transporter les VLANs entre switch et routeur.
configure terminal
interface fa0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30
end
write memory


Étape 4 : Sous-interfaces du Routeur 1

configure terminal

interface g0/0.30
 encapsulation dot1Q 30

Partie 2 — Serveur DHCP centralisé


Étape 1 : Connecter le serveur DHCP sur un VLAN dédié
enable
configure terminal
vlan 99
 name DHCP
interface fa0/10
 switchport mode access
 switchport access vlan 99
end
write memory


Étape 2 : Autoriser le VLAN DHCP sur le trunk
configure terminal
interface fa0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,99
end
write memory


Étape 3 : Sous-interface VLAN DHCP sur le routeur
configure terminal
interface g0/0.99
 encapsulation dot1Q 99
 ip address 192.168.99.1 255.255.255.0
end
write memory


Étape 4 : Activer le relais DHCP
configure terminal
interface g0/0.10
 ip helper-address 192.168.99.10
interface g0/0.20
 ip helper-address 192.168.99.10
interface g0/0.30
 ip helper-address 192.168.99.10
end
write memory

Partie 3 — Routage dynamique RIPv2


  

    Étape 1 : Activer RIPv2 sur le Routeur 1
    
Déclarer les réseaux connectés au Routeur 1 et désactiver l’auto-résumé.

    
enable
configure terminal
router rip
 version 2
 no auto-summary
 network 10.0.0.0
 network 192.168.10.0
 network 192.168.20.0
 network 192.168.30.0
end
write memory

  


  

    Étape 2 : Activer RIPv2 sur le Routeur 2
    
Déclarer les réseaux connectés au Routeur 2 et désactiver l’auto-résumé.

    
enable
configure terminal
router rip
 version 2
 no auto-summary
 network 10.0.0.0
 network 192.168.110.0
 network 192.168.120.0
 network 192.168.130.0
end
write memory

  


  

    Étape 3 : Rendre passives les interfaces LAN du Routeur 1
    
Empêcher l’envoi d’annonces RIP vers les VLANs utilisateurs.

    
configure terminal
router rip
 passive-interface g0/0.10
 passive-interface g0/0.20
 passive-interface g0/0.30
end
write memory

  


  

    Étape 4 : Rendre passives les interfaces LAN du Routeur 2
    
Empêcher l’envoi d’annonces RIP vers les VLANs utilisateurs.

    
configure terminal
router rip
 passive-interface g0/0.10
 passive-interface g0/0.20
 passive-interface g0/0.30
end
write memory

  


  

    Étape 5 : Vérifier les routes apprises
    
Contrôler la table de routage et les voisins RIP.

    
show ip route
show ip protocols

  





  
Partie 4 — Routage dynamique EIGRP


  

    Étape 1 : Activer EIGRP sur le Routeur 1
    
Créer le processus EIGRP AS 100, annoncer les réseaux locaux et désactiver l’auto-résumé.

    
enable
configure terminal
router eigrp 100
 no auto-summary
 router-id 1.1.1.1
 network 10.0.0.0 0.0.0.3
 network 192.168.10.0 0.0.0.255
 network 192.168.20.0 0.0.0.255
 network 192.168.30.0 0.0.0.255
end
write memory

  


  

    Étape 2 : Activer EIGRP sur le Routeur 2
    
Créer le même AS EIGRP sur le second routeur et annoncer ses réseaux locaux.

    
enable
configure terminal
router eigrp 100
 no auto-summary
 router-id 2.2.2.2
 network 10.0.0.0 0.0.0.3
 network 192.168.110.0 0.0.0.255
 network 192.168.120.0 0.0.0.255
 network 192.168.130.0 0.0.0.255
end
write memory

  


  

    Étape 3 : Rendre passives les interfaces LAN du Routeur 1
    
Bloquer l’envoi des paquets EIGRP vers les VLANs utilisateurs.

    
configure terminal
router eigrp 100
 passive-interface g0/0.10
 passive-interface g0/0.20
 passive-interface g0/0.30
end
write memory

  


  

    Étape 4 : Rendre passives les interfaces LAN du Routeur 2
    
Bloquer l’envoi des paquets EIGRP vers les VLANs utilisateurs.

    
configure terminal
router eigrp 100
 passive-interface g0/0.10
 passive-interface g0/0.20
 passive-interface g0/0.30
end
write memory

  


  

    Étape 5 : Modifier les temporisateurs Hello et Hold
    
Accélérer la détection de perte de voisinage sur le lien inter-routeurs.

    
configure terminal
interface g0/1
 ip hello-interval eigrp 100 2
 ip hold-time eigrp 100 6
end
write memory

  


  

    Étape 6 : Ajuster la bande passante du lien
    
Influencer la métrique EIGRP avec une valeur de bande passante cohérente.

    
configure terminal
interface g0/1
 bandwidth 100000
end
write memory

  


  

    Étape 7 : Activer l’authentification MD5
    
Sécuriser les échanges EIGRP sur le lien entre routeurs.

    
configure terminal
key chain EIGRP_KEYS
 key 1
  key-string MonSecret123

interface g0/1
 ip authentication mode eigrp 100 md5
 ip authentication key-chain eigrp 100 EIGRP_KEYS
end
write memory

  


  

    Étape 8 : Résumer plusieurs routes sur le Routeur 1
    
Annoncer un préfixe agrégé vers le voisin.

    
configure terminal
interface g0/1
 ip summary-address eigrp 100 192.168.0.0 255.255.0.0
end
write memory

  


  

    Étape 9 : Vérifier le voisinage et les routes
    
Contrôler les adjacences EIGRP et la table de routage.

    
show ip eigrp neighbors
show ip eigrp topology
show ip route eigrp
